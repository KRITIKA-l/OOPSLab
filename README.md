# OOPS Lab Repository
This repository contains all laboratory work completed as part of the Object-Oriented Programming (OOP) course. It showcases the step-by-step progression from basic C++ concepts to advanced OOP implementations, culminating in a fully functional mini-project.

## Purpose
The primary purpose of this repository is to organize and demonstrate the practical application of key OOP principles in C++. Each folder corresponds to a concept explored during lab sessions, helping strengthen object-oriented design, modular coding, reusability, and analytical problem-solving skills.

## Contents
- **Basics** – Introductory programs focusing on syntax, data types, conditionals, and loops.
- **Classes and Objects** – Implementation of constructors, objects, and member functions.
- **Inheritance** – Examples of single, multilevel, and multiple inheritance.
- **Polymorphism** – Programs demonstrating overloading, overriding, and virtual functions.
- **Exception Handling** – Use of try–catch blocks and custom exceptions.
- **Templates** – Function and class template implementations.
- **MINIPROJECT.cpp** – Complete implementation of *Virtual Human with Emotions*.

## Mini-Project: Key Features
- **Virtual Human Simulation** – Represents a human-like character with emotional and physical attributes.
- **Full OOP Architecture** – Uses abstraction, inheritance, polymorphism, and encapsulation effectively.
- **Two Distinct Roles** – Student and Employee, each with unique work behaviors.
- **Dynamic State Management** – Real-time updates to energy, hunger, stress, happiness, health, and social levels.
- **Mood Evaluation System** – Computes overall mood based on emotional parameters.
- **Time-Based Simulation** – Activities progress through morning, afternoon, evening, and night.
- **Role-Specific Polymorphism** – Student gains knowledge; Employee earns salary.
- **Text-Based Visual Indicators** – Clear bar-graph style representation of changing stats.
- **Robust Input Handling** – Prevents invalid input and ensures smooth interaction.
- **Modular & Extensible Design** – New characters, activities, or features can be added easily.
